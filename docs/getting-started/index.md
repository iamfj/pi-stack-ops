# Getting started

Start with the page that matches what you need right now.

| Need | Read | Outcome |
| --- | --- | --- |
| Install or verify tools | [Installation](installation.md) | You know which tools are required, which are optional, and what `stack-ops doctor` checks. |
| Understand the workflow boundary | [What is pi-stack-ops?](what-is-pi-stack-ops.md) | You know when to use Stack Ops and when not to. |
| Run the shortest executable path | [Quickstart](quickstart.md) | Local state exists, tools are checked, and a spec/plan flow is started. |
| See the full first stack | [Your first stack](first-stack.md) | You understand spec PR, slice PRs, validation, pause/resume, and PR preparation. |

If you are already inside a repository, the fastest safe path is:

```bash
stack-ops doctor
stack-ops init
stack-ops status
```

Then use Pi prompts from the [Quickstart](quickstart.md).
