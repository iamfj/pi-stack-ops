# Concepts

This section explains the ideas behind `pi-stack-ops`. Read it when you want to
understand why the workflow is shaped around specs, slices, local state, and
human checkpoints.

## Core ideas

The package uses a small set of concepts to keep AI-assisted stacked work
reviewable.

- [Stack Ops methodology](stack-ops-methodology.md) explains the end-to-end
  workflow.
- [Stacks and slices](stacks-and-slices.md) explains how large work becomes
  smaller pull requests.
- [Specs, ADRs, and plans](specs-adrs-and-plans.md) explains durable and
  disposable artifacts.
- [Agents and roles](agents-and-roles.md) explains how focused subagents support
  the workflow.
- [State and artifacts](state-and-artifacts.md) explains what lives under
  `.pi/stack-ops/`.
